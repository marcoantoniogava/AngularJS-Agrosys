# 🧠 Prototype e Prototype Chain (JavaScript real por trás dos objetos)

## 1. O que é Prototype?

Todo objeto em JavaScript tem uma **ligação interna invisível** chamada:

> `[[Prototype]]`

Isso significa que ele pode “herdar” coisas de outro objeto.

---

## 2. Exemplo simples

```javascript id="p1"
const pessoa = {
  nome: "Nícolas",
};

console.log(pessoa.toString());
```

Você nunca definiu `toString()`, mas mesmo assim funciona.

👉 Por quê?

Porque ele foi herdado do prototype.

---

## 3. Prototype na prática

Todo objeto em JS aponta para um “pai”:

```javascript id="p2"
const obj = {};
console.log(obj.__proto__);
```

Esse `__proto__` (antigo, mas útil pra entender) mostra o prototype.

---

## 4. Prototype Chain (cadeia de protótipos)

Quando você acessa algo:

```javascript id="p3"
obj.algumaCoisa
```

O JS faz isso:

1. procura no próprio objeto
2. se não achar → vai para o prototype
3. se não achar → vai para o prototype do prototype
4. até chegar no topo (`null`)

---

## 5. Exemplo visual da cadeia

```text
obj
 ↓
Object.prototype
 ↓
null
```

---

## 6. Criando herança com prototype

### Exemplo clássico:

```javascript id="p4"
function Pessoa(nome) {
  this.nome = nome;
}

Pessoa.prototype.falar = function () {
  console.log("Meu nome é " + this.nome);
};

const p1 = new Pessoa("Nícolas");

p1.falar();
```

---

## 7. O que aconteceu aqui?

* `p1` tem `nome`
* `falar()` não está dentro de `p1`
* então ele procura em:

```text
p1 → Pessoa.prototype → Object.prototype
```

E encontra.

---

## 8. Por que isso existe?

Porque economiza memória.

Se você fizesse assim:

```javascript id="p5"
function Pessoa(nome) {
  this.nome = nome;
  this.falar = function () {
    console.log(this.nome);
  };
}
```

👉 cada objeto teria sua própria função (ruim)

Com prototype:

👉 todos compartilham a mesma função

---

## 9. Prototype de funções

Funções também são objetos:

```javascript id="p6"
function teste() {}

console.log(teste.prototype);
```

👉 toda função construtora tem um `.prototype`

---

## 10. `new` por trás dos panos

Quando você faz:

```javascript id="p7"
const p1 = new Pessoa("Nícolas");
```

O JS faz isso internamente:

```javascript id="p8"
const p1 = {};

p1.__proto__ = Pessoa.prototype;

Pessoa.call(p1, "Nícolas");
```

---

## 11. Prototype chain real no dia a dia

Exemplo:

```javascript id="p9"
const arr = [1, 2, 3];

arr.map();
arr.filter();
```

Esses métodos vêm de:

```text
Array.prototype
```

---

## 12. Hierarquia completa

```text
Array → Array.prototype → Object.prototype → null
Function → Function.prototype → Object.prototype → null
Object → Object.prototype → null
```

---

## 13. Object.create (herança direta)

```javascript id="p10"
const pessoa = {
  falar() {
    console.log("oi");
  }
};

const p1 = Object.create(pessoa);

p1.falar();
```

👉 p1 não tem falar, mas herda do objeto pessoa

---

## 14. Shadowing (sobrescrita)

Se o filho tem a propriedade:

```javascript id="p11"
const pai = {
  nome: "pai"
};

const filho = Object.create(pai);

filho.nome = "filho";

console.log(filho.nome); // filho
```

👉 ele não usa o pai, porque encontrou primeiro nele mesmo

---

## 15. Como o JS resolve propriedades (regra mental)

Quando você faz:

```javascript id="p12"
obj.x
```

Ele faz:

```
1. obj
2. obj.__proto__
3. obj.__proto__.__proto__
4. null
```

---

## 16. Resumo brutal (pra fixar)

* Todo objeto tem prototype
* Prototype é outro objeto
* Se não acha algo → sobe na cadeia
* Funções construtoras usam prototype pra herança
* `new` conecta tudo automaticamente
* Array, Function e Object são todos baseados nisso

---
